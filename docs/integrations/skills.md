# Skills導入判断

この文書は、repo内にskillを追加すべきかどうかを判断するための補助文書です。
MCPのような外部接続の導入判断とは分けつつ、同じ粒度で比較できるようにします。

## いつ検討するか
- 同じ依頼手順を複数回説明しているとき
- 文書更新やレビュー観点が繰り返し登場するとき
- 単なる短いルール列では足りず、手順、例、補助スクリプト参照が必要なとき
- そのrepo固有の進め方を、以後のセッションでも再利用したいとき

## skillに向くもの
- 複数手順からなる定型workflow
- repo固有の文書読み順や判断基準
- 例、テンプレート、補助スクリプトを伴う作業
- 明示呼び出しでも暗黙選択でも価値がある作業

## skillに向かないもの
- 1行で済む恒久ルール
- `AGENTS.md` に短く書けば足りる指示
- 実際には1回しか使わない手順
- 外部接続や認証が本体で、workflow自体は薄いもの

## `AGENTS.md` とskillの切り分け
- `AGENTS.md`: 常に効かせたい短い恒久ルール
- skill: 条件に応じて使う再利用workflow

## MCPとskillの切り分け
- MCP: 外部システムへの接続やツール提供
- skill: そのツールをどういう順で使うかという手順
- 外部システム利用が必要でも、手順の再利用価値が高ければ `skill + MCP` を検討する

## 判断観点
- 再利用性: 別の依頼でも同じ流れで使えるか
- repo依存度: このrepoの文書や構造に強く依存するか
- 説明量: `AGENTS.md` だけでは収まらないか
- 保守コスト: 更新箇所が増えすぎないか
- 誤誘導リスク: 曖昧なdescriptionで意図しない場面でも呼ばれないか

## 導入の目安
- 同種の作業で2回以上ほぼ同じ指示を書いている
- 手順を固定した方がミスや読み過ぎを減らせる
- repo内に置いた方が、外部参照より判断が速い

## 記録先
- repo固有skillは `.agents/skills/<skill-name>/SKILL.md`
- skill追加の背景や補足判断は、必要ならこのディレクトリや ADR に残す
- 短い恒久ルールへ落とし込める部分は、必要に応じて `AGENTS.md` へ戻す
