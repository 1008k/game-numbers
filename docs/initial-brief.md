# 初期ブリーフ

この文書は `計算パネルパズル` の初期企画メモです。仕様の正本は `docs/project-spec.md`、デザイン方向は `DESIGN.md` を参照します。

## 1. 何を作るか
- プロジェクト名: 計算パネルパズル
- プロジェクト種別: スマホ向けカジュアルパズルゲーム
- 一言でいうと: 数字パネルと演算子パネルを消費して複数ターゲットを成立させる計算ロジックパズル
- 解決したい課題: 計算ドリル感を抑え、短時間で遊べるリソース管理型の数字パズル体験を作る
- 成功条件: 初見でルールが伝わり、タップだけで遊べ、複数ターゲットのパネル配分が面白さとして成立する

## 2. 誰のためか
- 主な利用者: 脳トレ、数字パズル、短時間カジュアルゲームを好むスマホユーザー
- 利用シーン: 移動中、休憩中、すきま時間
- 公開範囲: 一般公開想定

## 3. UI / デザインの方向
- ユーザーに感じてほしいこと: 分かりやすさ、手触りのよさ、正解時の気持ちよさ
- 参考にしたいUIやサービス: 電卓、数字タイル系パズル、短時間スマホゲーム
- 避けたい見た目や操作感: 計算ドリル感、数字入力の面倒さ、反応の遅さ、説明頼みのUI
- 最初に作りたい代表画面: ゲーム画面

## 4. 今回やること / やらないこと
- 必須機能:
  - ステージ制
  - ターゲット3個
  - パネル消費
  - 正誤判定
  - ミス3回
  - 広告復活
  - 解答から逆算するステージ生成
- 今回やらないこと:
  - ランキング
  - デイリー
  - 長い式
  - 特殊演算
  - 大量スキン
  - ストア公開手順の具体化

## 5. 制約
- 希望スタック: Godot 4.x / GDScript
- 既存システムや外部連携: MVPでは広告復活のみ検討
- 認証の要否: MVPでは不要
- データ保存の要否: ステージ進捗や設定のローカル保存を想定
- デプロイ先・運用環境: iOS / Android
- 納期や優先度: MVP仕様の具体化を優先
