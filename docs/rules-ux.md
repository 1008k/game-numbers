# UI/UXルール

共有ルールは `.shared/docs/rules-ux.md` を基本として参照します。

この文書は、`game-numbers` 固有の体験判断だけを残す入口です。画面ごとの仕様は `docs/project-spec.md`、持ち運ぶデザイン方向は `DESIGN.md` を正本にします。

## 体験の軸
- ユーザーが数値を手入力するゲームではなく、タイルを選んで組み合わせを作るゲームとして扱います。
- 1ステージ中はゲーム画面への集中を優先し、ホームや設定への導線は補助に留めます。
- 主要操作はタップによる `number operator number` の組み立てです。ドラッグ必須、長押し必須、キーボード入力中心の操作にはしません。
- 複数ゴールを同時に最適化する気持ちよさ、タイルが消費される納得感、正解時の手触りを重視します。

## 画面と状態
- Home は開始導線を主目的にし、初回でもそのまま始められる状態にします。
- Game はステージ情報、残りミス、ゴール、現在の組み合わせ、使用可能タイルを一目で追える構成にします。
- Clear は達成感と次のステージへの移動を優先します。
- Game Over は復活、リトライ、ホームへの復帰を迷わせない構成にします。

## フィードバック
- タイル選択、組み合わせ完成、正解、ミス、タイル消費は即時に見える形で返します。
- 正解済みゴール、選択中ゴール、未クリアゴールは視覚的に区別します。
- 消費済みタイルは、他ゴールで再利用できないことが分かる状態にします。
- ミス理由は短く伝え、次の操作を妨げない表示にします。

## レイアウト
- スマホ縦画面を基準にします。
- 数字と演算子は読みやすさを優先し、タイルの押せる領域を小さくしすぎません。
- 情報密度は、ゴール、式、タイル、補助操作の優先順位に従って調整します。
- 装飾はタイルボード感と達成感を支える範囲に留め、学習教材風の堅い印象や説明過多を避けます。

## 更新条件
- タイルパズルの中心トーン、タップだけで遊ぶ前提、縦固定の画面構成を変える場合は `DESIGN.md` も更新します。
- ランキング、デイリー、大量スキン、課金画面の詳細は MVP 外です。追加する場合は仕様または track に判断を残します。
