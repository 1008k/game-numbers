# Tracks

このディレクトリは、仕様、設計、運用、公開物に影響する大きめの変更を作業単位で残す場所です。
軽微な修正では必須にせず、背景、スコープ、主要判断、実施計画、完了条件を分けて残したいときに使います。

## 使いどころ
- 機能追加、まとまったリファクタリング、運用変更、公開フロー変更
- 途中で中断しても再開しやすくしたい作業
- 仕様と実装の間に短い計画を置きたい作業

## 基本方針
- 1つの変更群につき1ディレクトリ
- ディレクトリ名は `YYYYMMDD-short-title` のように日付と短い識別子で付ける
- 少なくとも `spec.md` と `plan.md` を置く
- 必要なら `notes.md` や補助資料を追加してよい
- 作業完了後、恒久ルールだけを `docs/project-spec.md` や `docs/rules-coding.md` へ反映する
- 利用者向けに共有すべき変更があれば、必要に応じて `CHANGELOG.md` から該当 `track` を参照する

## 最小構成
- `spec.md`: 背景、スコープ、主要判断、完了条件
- `plan.md`: 実施手順、確認項目、進捗
- `metadata.yaml`: 状態、関連PR、関連Issue、更新日などの軽い索引

## テンプレート
`0000-template/` をコピーして使います。
