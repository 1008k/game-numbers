# 計算パネルパズル仕様反映

## 背景
添付された `calc_panel_game_spec.md` を観測元として、汎用のdocs-firstテンプレートだったリポジトリを、スマホ向けGodotゲーム企画の文書群へ具体化する。

## スコープ
- `docs/project-spec.md` を仕様の正本として更新する。
- `DESIGN.md` に電卓風UI、タップ操作、即時フィードバックの方向を定義する。
- `README.md` と `ARCHITECTURE.md` を企画リポジトリ向けの入口と構造説明に更新する。
- `docs/rules-coding.md` にGodot 4.x/GDScript前提と責務分割を最小追加する。

## 主要判断
- プロダクト名は正式決定まで `計算パネルパズル` とする。
- MVPはGodot 4.x/GDScript、iOS/Android、縦固定、ターゲット3個を既定とする。
- MVPの式入力は `number operator number` のみとする。
- 広告SDK、課金、ランキング、デイリー、長い式、特殊演算はMVP後の拡張とする。

## 完了条件
- ターゲット3個、有限パネル、ミス3回、広告復活1回のMVPが仕様から読み取れる。
- 使用済みパネルを他ターゲットで再利用できないことが明記されている。
- 0除算、式不成立、数字不足、演算子不足、計算結果不一致がミス条件として定義されている。
- ステージ生成方針が「先に解答を作る」であることが明記されている。
