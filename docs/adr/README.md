# ADRs

このディレクトリは、重要な設計判断を短く記録するための場所です。
`docs/project-spec.md` が「何を作るか」、`ARCHITECTURE.md` が「どう構成されているか」を扱うのに対し、ADR は「なぜその判断にしたか」を残します。

## 使いどころ
- 複数案があり、どれを選んだかを後から説明したいとき
- 将来見直す可能性が高い技術判断を残したいとき
- エージェントや新しい参加者が、過去の判断理由を素早く把握したいとき

## 書き方
- 1件につき1ファイル
- ファイル名は `0001-short-title.md` のように連番付きにする
- 長文の議論ではなく、背景、選択肢、決定、影響を短く残す

## 最小テンプレート
`0000-template.md` をコピーして使ってください。
