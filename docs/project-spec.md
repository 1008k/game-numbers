# プロジェクト仕様書

この文書は `ナンバータイル` の要件定義と仕様の正本です。参照先や優先順位は `docs/policy-index.yaml` を先に確認し、実装判断は `docs/rules-coding.md`、UI/UX判断は `docs/rules-ux.md` と `DESIGN.md` を参照します。

## 1. 目的
- 数字タイルと記号タイルを使って、複数のゴール数字を成立させる脳トレ系カジュアルパズルを作る。
- 中心体験は、計算問題を解くことではなく、限られた数字と記号をどのゴールへ配分するかを考えるリソース管理型ロジックパズルとする。
- 1ステージを短時間で遊べ、タップだけで操作でき、数字タイルを盤面へ割り振る気持ちよさを持つゲームを目指す。
- 広告付きスマホカジュアルゲームとして成立する構成を前提にする。

## 2. 対象とプラットフォーム
- 主な利用者: 短時間で遊べる数字パズル、脳トレ、ロジックパズルを好むスマホユーザー。
- 対応プラットフォーム: iOS、Android。
- 使用エンジン: Godot 4.x。
- 使用言語: GDScript。
- 画面方向: 縦固定。
- 公開範囲: 一般公開を想定するが、ストア公開手順はMVP後に具体化する。

## 3. コア体験
- 見た瞬間にルールが理解できる。
- 少し考えると解ける。
- 複数ゴールを同時に最適化する感覚が気持ちよい。
- 数字入力ではなく、タイルを選ぶだけで組み合わせを作れる。
- 正解時の反応が速く、手触りがよい。

## 4. MVP範囲
- ステージ制。
- 1ステージあたりゴール3個。
- 数字タイルと記号タイルの消費制。
- 式入力は `number operator number` のみ。
- 演算子は `+`, `-`, `×`, `÷`。
- 正誤判定。
- ミス回数制限。
- リワード広告による復活。
- 解答から逆算するステージ自動生成。
- スマホ縦画面対応。

## 5. 基本ルール
- 画面上部に複数のゴール数字を表示する。
- 画面下部に数字タイルと記号タイルを表示する。
- プレイヤーはゴールを1つ選び、数字、記号、数字の順にタイルを選んで組み合わせを作る。
- 組み合わせの計算結果が選択中ゴールと一致したら、そのゴールをクリアする。
- 正解した組み合わせで使用したタイルは消費され、他ゴールでは再利用できない。
- すべてのゴールを成立させるとステージクリア。

## 6. タイル仕様
- 数字タイルは1桁固定ではなく、難易度に応じて1桁、2桁、3桁を扱う。
- MVPでは通常の整数タイルを中心に扱う。
- 分数、小数、負数は上級ステージ向けの将来拡張とし、MVPでは必須にしない。
- 分数を導入する場合は、分数入力UIではなく完成済みの数字タイルとして扱う。
- 記号タイルはMVPでは `+`, `-`, `×`, `÷` の4種とする。

## 7. ミスとゲームオーバー
- MVPの最大ミス回数は3回。
- 次の条件でミスを1回消費する。
  - 計算結果がゴールと一致しない。
  - 式が成立していない。
  - 必要な演算子が不足している。
  - 必要な数字タイルが不足している。
  - 0除算が発生する。
- 残りミスが0になったらゲームオーバーにする。
- ゲームオーバー時は、リトライ、ホームへ戻る、広告で復活を提示する。
- リワード広告による復活は、MVPでは1ステージ1回までとし、お手付き回数を+1して現在ステージを継続する。

## 8. ステージ設計
- 初級: 足し算中心、1桁数字、ゴール2個から導入する。
- 中級: 2桁数字、掛け算、ゴール3個を扱う。
- 上級: 割り算、ダミータイル、ゴール4個を扱う。
- 超上級: 小数、負数、長い式、時間制限、最適化重視の要素を候補にする。
- MVPの基準ステージは、ゴール3個、通常整数、四則演算、ミス3回とする。

## 9. ステージ生成
- ステージは先に解答を作り、そこからゴールとタイルを生成する。
- 例: `3 × 4 = 12`, `5 + 2 = 7`, `24 ÷ 3 = 8` のような解答群を先に作る。
- 解答に使った数字と演算子をタイル化し、必要に応じて難易度に応じたダミータイルを追加する。
- この方針により、解なし問題や理不尽な問題を避ける。
- 複数解は許容し、どの解法を選ぶかをゲーム性に含める。

## 10. 画面仕様
- ホーム画面:
  - スタート。
  - 設定。
  - MVP後の候補としてデイリー、広告削除を追加できる。
- ゲーム画面:
  - ステージ情報。
  - ゴール数字。
  - 現在の式。
  - タイルエリア。
  - 機能ボタン。
- クリア画面:
  - クリア演出。
  - スコア。
  - 次へ。
  - リワード広告。
- ゲームオーバー画面:
  - リトライ。
  - ホームへ戻る。
  - 広告で復活。

## 11. UX方針
- 操作ストレスを減らすことを最重要にする。
- タップだけで遊べる。
- 数字入力を不要にする。
- 操作ごとに即時フィードバックを返す。
- 正解演出を強くし、タイル吸着、ゴール点灯、コンボ演出、正解時エフェクトを候補にする。
- 詳細な視覚方向は `DESIGN.md` を正とする。

## 12. データ構造

### StageData
```gdscript
class_name StageData

var targets: Array
var panels: Array
var solutions: Array
var difficulty: int
var max_mistakes: int = 3
```

### PanelData
```gdscript
class_name PanelData

var display_text: String
var value
var type: String
var used: bool = false
```

### 補足
- `targets` はステージで成立させるゴール数字の配列。
- `panels` は数字タイルと記号タイルの配列。
- `solutions` は生成時に作った成立式を保持する。
- `difficulty` はステージ生成と表示上の難易度に使う。
- `used` はタイル消費状態を表す。

## 13. マネタイズ
- インタースティシャル広告は数ステージごと、または失敗時に低頻度で表示する候補とする。
- リワード広告は復活、ヒント、シャッフル、ボーナスに利用する候補とする。
- 課金は広告削除、テーマ変更、タイルスキンを候補とする。
- MVPで必須なのは広告復活のみとし、広告SDKや課金導入の詳細は実装段階で別途決める。

## 14. MVP対象外
- デイリー固定問題。
- ランキング。
- 特殊演算子。
- 長い式。
- 大量のスキン。
- PvP。
- 無限モード。
- イベント。
- 世界ランキング。
- AI問題生成。
- ストア公開手順、分析基盤、広告SDK詳細、課金実装。

## 15. タイトル候補
- 日本語: 計算パネル、数式パズル、けいさんロジック、電卓パズル、数字パネル。
- 英語: Calc Panel、Number Logic、Formula Tiles、Calc Puzzle、Number Formula。
- 旧作業名は `計算パネルパズル`。実装とユーザー向け文書では `ナンバータイル` を使う。

## 16. 受け入れ条件
- 仕様から、ゴール3個、有限タイル、ミス3回、広告復活1回のMVPが読み取れる。
- 使用済みタイルを他ゴールで再利用できないことが明確である。
- 0除算、式不成立、数字不足、演算子不足、計算結果不一致がミス条件として定義されている。
- ステージ生成方針が「先に解答を作る」であり、解なし問題を避ける設計になっている。
- MVP対象外が明記され、長い式、括弧、特殊演算、分数入力UIを先回り実装しない判断ができる。
- Godot 4.x/GDScript/iOS/Android/縦固定という実装前提が確認できる。
