# 文書表記ルール

共有ルールは `.shared/docs/rules-writing.md` を基本として参照します。

この文書は、`game-numbers` 固有の表記判断だけを残す入口です。優先順位は `docs/policy-index.yaml`、仕様用語は `docs/project-spec.md`、利用者向け入口は `README.md` を参照します。

## プロジェクト用語
- プロダクト名は `ナンバータイル` を基本にします。
- 主要要素は `ゴール`、`数タイル`、`記号タイル`、`タイル`、`ステージ`、`ミス`、`復活` を使います。
- 実装上の構造名は `StageData`、`PanelData`、`GameManager.gd`、`StageGenerator.gd`、`ExpressionEvaluator.gd` など、既存表記に合わせます。
- `panel` と `tile` のどちらを使うか迷う場合、ユーザーに見える説明は `タイル`、内部構造や既存クラス名は既存コードの表記を優先します。

## 文書ごとの役割
- `docs/project-spec.md` は、要件、MVP範囲、データ構造、受け入れ条件の正本です。
- `DESIGN.md` は、画面トーン、コンポーネント、Stitch などに渡すデザイン方向の正本です。
- `ARCHITECTURE.md` は、シーン、スクリプト、責務分割、外部連携方針の正本です。
- `README.md` は、概要、導線、ローカル実行方法に絞ります。
- `CHANGELOG.md` には利用者向けの動作変更だけを記録します。

## 表記の注意
- 仕様や設計で相対的な時間表現を使う場合は、必要に応じて基準時期を添えます。
- 実測していない難易度、継続率、広告効果、収益見込みは断定しません。
- 文字化けしている既存文書を直す場合は、意味を推測で広げず、仕様正本や実装から確認できる範囲で修正します。
